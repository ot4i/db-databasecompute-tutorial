Databases - Using a Compute node to insert data into a DB2 database via ODBC

IBM Knowledge Center
Learn how to use Compute node ESQL to insert data into DB2 using a message flow driven by a JSON message over HTTP.

Import projects

When you click Import, one project will be created in your workspace: An application named ExampleDatabaseCompute.

ExampleDatabaseCompute provides a single message flow named DatabaseCompute which uses an HTTPInput node to receive a JSON input message, a Compute node to communicate with DB2, and an HTTPReply node.

Once imported using the link on the right, the provided project is ready to be deployed and tested. The project will be deployed to the integration server using a provided BAR file. Before you do this, you will need to create the required DB2 database, as described on the Prepare tab.

Prepare the database, set up your credentials and deploy

This tutorial uses DB2 to create a database, which the App Connect Enterprise message flow will interact with. It is assumed you have DB2 already installed successfully, and that you have access to a userid which has the required level of DB2 administrative privileges. Create a new example database for this tutorial named USERS:

  1. Start a DB2 Command Window and execute the command:
    db2 create database USERS
  2. Next we will connect to the database which was just created and create a table. From the same window, start an interactive command session by typing db2 and then execute these commands:
    CONNECT TO USERS
    CREATE TABLE MYSCHEMA.EMPLOYEES (PKEY INTEGER NOT NULL, FIRSTNAME VARCHAR(30), LASTNAME VARCHAR(30), COUNTRY VARCHAR(2), PRIMARY KEY(PKEY))

Set up an ODBC connection in the operating system:

  1. This tutorial assumes you are using Windows, but if this is not the case, full instructions are provided for configuring ODBC in the Knowledge Center links at the base of this section. On Windows, open the ODBC Data Source Administrator (64-bit), switch to the System DSN tab, click the Add button, and select a driver for your database.
  2. Using DB2 in a default set-up, you should find an entry in the list labelled IBM DB2 ODBC Driver - DB2COPY1. Select this entry and click Finish.
  3. In the resulting Add dialog, set the Data source name = USERS, leave the Database alias = USERS and click OK. You will be returned to the ODBC Data Source Administrator (64-bit) window and you should clikc OK again to close this window too.

Create an integration node, set up your credentials and deploy by completing the following steps:

  1. Start an App Connect Enterprise command console and execute the following commands:
    mqsicreatebroker TESTNODE
    mqsistart TESTNODE
    mqsicreateexecutiongroup TESTNODE -e server
    mqsistop TESTNODE
    mqsisetdbparms TESTNODE -n odbc::USERS -u db2admin -p pass123
    mqsistart TESTNODE
  2. The above mqsisetdbparms command configures the runtime with the required credentials to access the USERS database.
  3. Drag and drop deploy the provided BAR file named DatabaseCompute.bar which you will find inside the ExampleDatabaseCompute application.

Use the Flow Exerciser to test the tutorial scenario

  1. Open DatabaseCompute.msgflow.
    • Click the Flow Exerciser icon to start testing the flow (when challenged you do not need to redeploy the flow again).
    • Click the Send Message icon .
    • Five saved messages are provided. Each message will drive the message flow to insert a row into the database table. Each message provides a JSON structure which has an equivalent field for each column in the database table. Select InputMessage1 and click Send and your message is sent to the HTTPInput node.
    • InputMessage1 contains this data:
      {"PrimaryKey": 1,"FirstName":"Ben","LastName":"Thompson","Country":"GB"}
    • After you close the dialog, the paths taken through the messageflow are highlighted. Repeat the test with the other four input messages, then check the database to see the data has been inserted successfully.

This tutorial has shown the Compute node using ESQL to communicate with a DB2 database using ODBC in order to insert data.
